# ComelyMD - 高阶轻量级 Markdown 分享系统

一个极简、低资源占用且易于在各类云服务器上自动化部署的 Markdown 独立分享工具。基于 `Go + SQLite + Pico.css` 构建。在提供稳定可靠的文本托管呈现基础上，融合了强大的自定义隐私与生命周期控制方案。

## ✨ 核心特性

- **极低的负载**：通过单一 Go 静态执行文件支撑，结合纯 `Go (modernc)` 移植的 CGO-Free SQLite 驱动。
- **免维护隔离**：无文章列表聚合，不提供全局导航暴露，通过独立专属高熵 ID (`Base62`) 保障分享环境天然隔离。
- **安全的解析**：内嵌 `Goldmark` 以及强化定制策略后的 `bluemonday` 驱动，强制拦截所有针对网页的 XSS 恶意脚本注入。
- **优雅的排版**：脱离重型构建，前端依托于 `Pico.css` 核心网格以及 `highlight.js` 实现美观自适应的代码与文本结构阅读。
- **高级权限锁**：支持开启四位无序哈希随机暗语校验，作为获取受保护便签的独立查核通行证屏障。
- **生命周期控制**：提供从精确至 5 分钟、长效至近一月，甚至是“阅后即刻焚毁粉碎”等多重自动化回收清理策略。

## 🚀 部署指南

推荐使用附带的 `docker-compose.yml` 环境进行极其快速的一键式镜像封装与启动部署：

### 快速安装与运行

```bash
# 1. 克隆项目源代码并进入当前工作目录
git clone https://github.com/Loxonl/comelyMD.git
cd comelyMD

# 2. 从本地源码构建并后台启动容器服务
docker-compose up -d
```

默认环境下，容器会在本地建立关联映射并对外暴露至 `18080` 端口。配合 Nginx 或 OpenResty 等反向代理工具指向 `http://127.0.0.1:18080` 即可对外网安全提供分享服务。数据统一持久化挂载保护于工程的 `/data` 目录。

> **服务更新说明**：当需要进行版本升级或获取最新社区代码时，重新运行 `docker-compose up -d --build` 进行源码重建，即可完成新旧容器和特性的平滑替换。

## 🛠️ 后端 API 接口

本项目可剥离前端被作为纯粹的后端解析生成接口层，供第三方系统与跨平台服务接入调用：

- `POST /api/pages`
  - 请求载荷 (`FormData`)：
    - `markdown`: (必备) 传入需要分享的 Markdown 正文内容
    - `is_burn`: (可选) 文本是否设为单次访问后即焚销毁："true" 
    - `expire_time`: (可选) 设定内容回收生命周期时长（枚举值："5m", "1h", "6h", "24h", "7d", "30d"）
    - `with_password`: (可选) 开启并分配四字符高强度核查锁："true"
  - 响应示例：`{"id": "xY8b2K", "url": "https://[对外绑定的分享域名]/p/xY8b2K", "pwd": "kA3P"}`

## 📂 底层源码边界说明

* `/data`: SQLite 数据库系统环境持久化及自动恢复挂靠目录。
* `/render`: Markdown 转译、标签转换支持及防护注入消毒引擎模块组件。
* `/handler`: 路由节点分发网关（统筹前端页面视图模型渲染与接收提取表单业务数据）。
* `/storage`: SQLite 底层存储器支撑，囊括防冲突的 `Base62` 重载分配以及密令生成的分配中心。

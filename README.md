# ComelyMD - 最小可用版 Markdown 分享服务

一个极简、低资源占用且易部署的 Markdown 独立分享工具。设计初衷是为个人技术提供简单便捷的无列表防爬文本共享与即焚短链转化呈现能力，基于典型的 `Go + SQLite + Pico.css`。

## ✨ 核心特性支撑

- **极低的负载**：单一 Go 静态执行文件支撑，结合纯 `Go (modernc)` 移植的 CGO-Free SQLite 驱动。
- **免维护隔离**：无文章列表聚合，不设全局导航，通过独立专属高熵 ID (`Base62`) 保障分享天然隔离环境。
- **安全的解析**：内嵌 `Goldmark` 以及强化定制策略后的 `bluemonday` 驱动进行强制 XSS 恶意脚本段的拦截。
- **优雅的排版**：脱离重型构建使用纯前端通过 `Pico.css` 核心网格以及 `highlight.js` 实现美观自适应阅读。
- **容器化首选**：完美贴合 `OpenResty / 1Panel` 环境配置，支持 `Docker-Compose` 一键挂载启动隔离网络。

## 🚀 极其易用的部署方案

如果你已经拥有了运行支持的 Docker 与 Docker-compose 的环境，只需不到两分钟极小镜像即可完成全部运行部署：

```bash
# 1. 下载拥有当前项目全源文件后跳转切入对应的服务执行管理根目录
cd comelyMD

# 2. 调用命令给予 Docker 构建该分享应用，并让其分配后台启动与持久化挂载保护生成系统
docker-compose up -d
```

默认容器对外建立暴露接口至 `18080` 端口，即通过访问承载本系统的 `http://127.0.0.1:18080` (或配合 OpenResty 的域名重发代理) 即可开始创建分享内容！

## 🛠️ 后端 API 接口资源

如果你想将其作为纯粹的后端生成器通过别的平台接管：

- `POST /api/pages`
  - 提交参数 (FormData)：`markdown` (传入分享片段正文)
  - 成功返回：`{"id": "xY8b2K", "url": "http://服务地址/p/xY8b2K"}`

## 🛣️ 路线图 (Roadmap)

- [ ] 特性补充：为短链构建支持阅读即焚与到期自动化删除等限时有效操作。

## 📂 结构说明

* `/data`: SQLite 数据库挂载持久化处。
* `/render`: Markdown 转译及蓝月亮阻击消毒机制。
* `/handler`: 分发网关（统筹处理视图下放和业务表单数据）。
* `/storage`: SQLite 底层调度及高度防干扰的 `Base62` 短地址驱动。
